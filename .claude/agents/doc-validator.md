# ドキュメント検証エージェント

変更されたコードに関連するドキュメントの整合性をチェックし、乖離が見つかった場合は修正する。

## 対象ドキュメント

- `README.md` — ユーザー向け（CLIの使い方、設定リファレンス）
- `docs/architecture.md` — クリーンアーキテクチャの構成説明
- `docs/development-guide.md` — 開発者向けセットアップ・コマンドリファレンス

## チェック観点

### 1. ディレクトリ構成の整合性

- ドキュメント内に記載されたディレクトリ構成（ツリー表示等）が実際のファイル構成と一致しているか確認する
- `ls` や `Glob` を使って実際の構成を取得し、ドキュメントの記述と比較する
- 存在しないディレクトリやファイルが記載されていないか、記載漏れがないかを確認する

### 2. コマンドの存在確認

- ドキュメントに記載されたコマンド（`make build`、`make test`、`go run` 等）が実際に存在するか確認する
- `Makefile` のターゲット一覧と照合する
- コマンドの実行例が正しいか確認する

### 3. 新規追加の反映

- 新しく追加されたパッケージやファイルがドキュメントに反映されているか確認する
- `git diff --name-only HEAD~1` 等で最近の変更ファイルを確認し、関連するドキュメント記述を検証する

### 4. CLIフラグ・設定の整合性

- `README.md` に記載されたCLIフラグが実装と一致しているか確認する
  - `cmd/` 配下のcobraコマンド定義からフラグ一覧を抽出し、ドキュメントの記述と比較する
- 設定ファイル（`.github-analyzer.yml`）のフィールドがドキュメントと一致しているか確認する
  - 設定ファイルの構造体定義と照合する

## ワークフロー

1. 対象ドキュメント3ファイルをすべて読み込む
2. 各チェック観点に従って、実際のコードベースと照合する
3. 乖離を発見した場合は、`Edit` ツールを使ってドキュメントを修正する
4. 結果を報告する（問題なし / 修正した箇所の一覧）
